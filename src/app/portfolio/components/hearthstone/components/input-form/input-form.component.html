Angular Explanation

In this small app, the input form, and the output are separate components. Rather than two-way binding the input parameters to a model within the output component, I am using one-way binding to avoid any object links by reference.

When the input form information is submitted, a call is made to the appropriate HearthstoneService endpoint (which consumes the RapidAPI Hearthstone API). Once the data is returned, the input form emits an event with the new card information, which is passed up to the main controller. The main controller then passes this new collection down to the output component.

This new collection is one-way bound, and so will not change until the user has submitted a new request through the input form.
